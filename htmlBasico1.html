¿Qué es HTML?

!DOCTYPE define el tipo de documento (html, xml, etc). Es necesario para que el navegador muestre la página correctamente.

El texto entre los tags "html" describe la página web en sí.

El texto entre los tags "body" es el contenido visible de la página.

HTML "Lenguaje de marcado de hipertexto". Un lenguaje de marcado ("ml") es un cjto de "tags" de marcado

Las tags o etiquetas describen el contenido del documento. Los documentos HTML tienen tags y texto plano. También se les llama "páginas web". Los tags normalmente van por pares de apertura (<tag>)y de cierre (</tag>). Los elementos HTML son lo que hay entre dos tags. (Aunque a veces se llama igual tags que elementos para ambos). Siempre usar tags en minúsculas.

Los tags de bloque o elementos de bloque suelen empezar en una línea nueva en el navegador. Son, por ejemplo, las cabeceras "h", párrafos "p", listas "ul,ol", tablas "table", etc.
Los tags "inline" (en línea o dentro de línea) se muestran en el navegador sin saltar de línea. Ejemplos: Modificadores de texto como negrita "b", cabeceras o casillas de tablas "th ó td", enlaces o anchors "a", imágenes "img", etc.

Las cabeceras:

Para HTML 5: <!DOCTYPE html>

Para HTML 4.01: <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"> (o strict en su caso).

Para XHTML 1.0: <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

Partes del tag <head>

Tiene cosas del documento que no se ven. Puede contener estos tags:

<title>, <base,>, <link>, <meta>, <script>, <style>.

Tag <title>: Define el título de la página en el navegador, al añadirla a favoritos y en los motores de búsqueda. Uso: <title>"texto del título"</title>

Tag <base>: Define un objetivo o dirección web por defecto para todos los links de la página. Uso: <base href="http://www.w3schools.com/images/" /> ó <base target="_blank" />

Tag <link>: Define una relación o vínculo entre el documento html y un recurso externo. Se usa mucho para vincular o referenciar las hojas de estilo o style sheets, css. Uso: <link rel="hoja de estilo" type="text/css" href="mi estilo.css" />

Tag <style>: Define información del estilo del documento html. Es decir, de cómo se verá cada uno de los tags tipo. Uso: <style type="text/css"> body(background-color: yellow) p(color: blue)</style>

Tag <meta>: Define información sobre el propio documento html, como codificación de caracteres, palabras clave para motores de búsqueda, autor del documento, cuándo se modificó por última vez, etc.
Uso: Descripción de la página: <meta name="description" content="Free Web tutorials on HTML, CSS, XML" />
Palabras clave para motores de búsqueda y tal: <meta name="keywords" content="HTML, CSS, XML" />

Tag <script>: Define un script del lado del cliente, como por ejemplo un JavaScript.

Formas de usar CSS (Cascade Style Sheets) en documentos HTML:

En línea para un sólo tag, con el atributo "style" de los tags

En la cabecera para todo el archivo HTML, con el tag <style>

Con archivos CSS que sirven para definir el estilo de varios documentos HTML.


Estilos en línea: Ejemplo: <p style="color: blue; margin-left: 20px;">Esto es un párrafo con estilo</p>

Esto es un párrafo con estilo

Hay muchos atributos de estilo como background-color, font-family, font-size


Estilo en el documento HTML (Style Sheet (hoja de estilo) interna al documento): Ponemos instrucciones CSS dentro del tag <head> con el tag <style>. Ejemplo:

<head>
<style type="text/css">
body {background-color: yellow;} p {color: red;} </style>
</head>

Style Sheet (hoja de estilo) externa: Vinculamos una CSS al archivo HTML, dentro del tag <head> con el tag <style>. Ejemplo:
<head>
<link rel="stylesheet" type="text/css" href="miestilo.css" />
</head>

name: "Nombre del mapa": Lo asocia con el elemento <img> cuyo atributo "usemap" coincide con "name".
Los tags <area> son "trozos" delimitados del mapa, y clickables.

Tag <area>: Define y delimita una región clickable de una imagen <map>.
Uso:
<area shape="rect" coords="0,0,52,14 href="dirección web o JavaScript" alt="..." /> Rectángulo (2 esquinas)
<area shape="circle" coords="0,0,52 href="dirección web o JavaScript" alt="..." /> Círculo (Centro y radio)
<area shape="poly" coords="0,0,52,14,21,98 href="dirección web o JavaScript" alt="..." /> Polígono (vértices)
Atributos:
shape: "Tipo de figura": rect, circle, poly.
coords: "Coordenadas necesarias para delimitar la figura definida en "shape"":
rect: coordenadas de esquinas NO y SE; circle: coordenadas del centro y longitud del radio;
poly: coordenadas de los vértices
href: "link (enlace) o JavaScript" que se activa al clickar en ese <area>
alt: "Texto informativo (hint de la imagen) o alternativo si no se ve"

El tag <div>

Es un tag o elemento de bloque que puede contener a otros. Junto con CSS podemos usarlo para darles atributos de estilo a los elementos que contiene. ¡OJITO! Qué cosa tan rara!! Sirve tb para definir el "layout" o la "disposición espacial" del documento. Se supone que el correcto uso de <table> no es para hacer tablas, sino para mostrar datos tabulados. El tag <span>

Es un elemento o tag en línea que se usa como contenedor de texto. Si se usa junto con CSS, nos sirve para darle atributos de estilo a partes del texto.

URL's

Significa: Uniform Resource Locator. Cada documento como documentos(páginas web, etc) tiene su propa URL, que se puede expresar bien en palabras con el nombre del sitio web: www.pepe.com o en dirección IP: 192.168.21.15. Ambos son sinónimos.

Una URL (en palabras) se compone de:
esquema://host.dominio:puerto/ruta/nombre_archivo
Sus partes son:

  • scheme o esquema: define el tipo de servicio de Internet. Hay varios servicios:
    • http (HiperText Transfer Protocol): El más habitual; páginas web normales, no encriptadas.
    • https (Secure HiperText Transfer Protocol): Páginas web seguras, encriptadas.
    • ftp (File Transfer Protocol): Para cargar o descargar archivos, sirve para mantener los dominios.
    • file: Ruta de un archivo en tu computador.
  • host: Define el dominio anfitrión, por defecto para el servicio http es www.
  • domain o dominio: Define el nombre del dominio de Internet donde se encuentra la página, como w3schools.com.
  • :port o :puerto: Define el número de puerto en el host (en el servicio http por defecto es 80).
  • path o ruta: Define una ruta en el dominio. Si se omite, estamos trabajando en el directorio raíz del dominio o sitio web.
  • nombre_archivo: Define el nombre del documento o recurso.

Problemas con la codificación en URL: No se permiten tildes, eñes, espacios en blanco, etc. El espacio en blanco es %20.
