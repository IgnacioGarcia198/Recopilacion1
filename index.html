Tags "input": Esto ha cambiado un montón ahora con HTML5, tiene un montón de atributos nuevos, ponemos todos:

  • "accept": Tipos de archivos que el servidor acepta. Se usa sólo con type="file".
  • "alt": texto alternativo.
  • "autocomplete": on/off, si se autocompleta el texto en el input o no.
  • "autofocus": si un elemento input debe tomar el foco nada más cargarse la página o no.
  • "checked": Para checkbox y radio, elementos "input" marcados por defecto.
  • "disabled": elemento input desabilitado.
  • "form": define el O LOS formularios a los que pertenece el elemento input.
  • "formaction": url del archivo que procesará el contenido del input al enviarlo (se usa en inputs de tipo "submit" e "image".
  • "formenctype": cómo se codifica la información para enviarla al servidor.
  • "formmethod": get o post, método HTTP para enviar los datos.
  • "formnovalidate": no validar los elementos para enviarlos.
  • "formtarget": dónde se muestra la respuesta recibida tras enviar la forma.
  • "height": altura de un elemento input tipo "image".
  • "list": id de un elemento "datalist" que tiene opciones predefinidas para un elemento input.
  • "max": Máximo valor admisible en un elemento input, para números o fechas.
  • "maxlength": Máxima longitud permitida en caracteres.
  • "min": Mínimo valor admisible, para números o fechas.
  • "multiple": Si se pueden meter varios valores en un elemento input.
  • "name": nombre del elemento input.
  • "pattern": expresión regular que debe cumplir la entrada del input.
  • "placeholder": pista corta sobre el valor esperado en el elemento input.
  • "readonly": input de sólo lectura.
  • "required": input que hay que rellenar obligatoriamente antes de enviar.
  • "size": ancho en caracteres del input.
  • "src": url de la imagen para usar como botón "submit", sólo en type "image".
  • "step": intervalos de números permitidos en un input.
  • "type": tipo de elemento input:
    • button

  • "value": valor de un elemento input.
  • "width": ancho de un elemento input, solo para type="image".

Aquí termina el resumen con todos los tags "extraños" de HTML. A continuación van los ATRIBUTOS GLOBALES que se aplican generalmente a todos los tags de HTML. Esto es para HTML5.

  • "accesskey": tecla que activa o da el foco a ese elemento.
  • "class": clase o clases de CSS a las que pertenece el elemento.
  • "contenteditable": si es "true", el contenido del elemento es editable, y si es "false", no.
  • "contextmenu": define el menú que aparece al hacer click derecho sobre el elemento. Da igual, no funciona en ningún navegador.
  • "dir": dirección del texto en el contenido del elemento, "rtl" o "ltr".
  • "dropzone": dice qué es lo que se debe hacer con un elemento arrastrado dropeado sobre nuestro elemento (copy, etc). No funciona en ninguno.
  • "hidden": el elemento se esconde porque no es relevante ahora mismo o lo que sea. No funciona en IE.
  • "id": La identificación unívoca de ese elemento.
  • "lang": lenguaje del contenido del elemento. Se usa como lang="fr".
  • "spellcheck": si "true", se revisa la gramática y palabras de su contenido. Sólo funciona en Chrome y Safari Xp
  • "style": estilo en línea para el elemento, de toda la vida.
  • "tabindex": número de orden del elemento cuando nos desplazamos entre distintos elementos con la tecla "tab".
  • "title": información extra o "hint" sobre el elemento, que se muestra cuando ponemos el puntero del ratón sobre el elemento.
